# UX Iteration 3 — 현재 시간표와 자동완성 후보 분리

## 문제

기존 `선택한 과목` 목록은 격자에 배치된 과목, 자동완성에만 쓰는 예비 과목, 제외 과목을 한 공간에 함께 표시했다. 역할 라벨은 존재했지만 사용자가 다음을 즉시 구분하기 어려웠다.

- 지금 시간표에 실제로 놓인 과목
- 아직 배치하지 않고 조합만 검토할 과목
- 자동완성에서 제외한 과목

또한 일반 과목 검색은 분반을 먼저 선택해야 했기 때문에, 단지 선택 과목 후보를 담으려는 상황에도 불필요한 결정을 요구했다.

## 결정

새 최상위 내비게이션을 추가하지 않고 `과목` 책임 안에서 다음 두 공간을 분리한다.

1. **현재 시간표** — 격자에 실제 배치된 과목과 자동 조정 범위
2. **자동완성 후보** — 시간표에는 놓지 않고 생성 알고리즘에만 전달할 과목

제외 과목은 정상 흐름을 차지하지 않도록 후보 영역 아래의 접힌 목록에서 관리한다. 후보는 특정 분반이 아니라 과목 단위로 담고, solver가 해당 과목의 공식 분반 전체에서 조건에 맞는 분반을 선택한다.

## 반응형 배치

- 390px 모바일: `자동완성` dialog의 첫 영역에 후보 바구니를 둔다. `후보 과목 담기`를 누르면 자동완성 dialog를 닫고 과목 검색 sheet로 전환해 겹친 modal을 만들지 않는다.
- 768–1199px: 오른쪽 자동완성 aside의 첫 영역에 둔다.
- 1200px 이상: 왼쪽 과목 aside에서 `현재 시간표` 다음에 둔다. 오른쪽 자동완성 영역에는 조건과 생성 결과만 유지한다.

## 구현 결과

- 후보 전용 과목 검색 제목과 설명 추가
- 검색 결과에서 분반 선택 없이 `후보로 담기` 제공
- 이미 현재 시간표나 후보에 있는 과목은 중복 담기 방지
- 후보를 `시간표에 넣기`, 삭제, 추가하는 직접 행동 제공
- 현재 시간표 목록에서 후보·제외 항목 제거
- 제외 과목은 접힌 보조 목록에서 해제 가능
- 라이트·다크 테마 토큰과 기존 44px 터치 목표 유지

## 검증

- Vitest: 후보 바구니와 과목 단위 담기 회귀 테스트 포함 86개 통과
- Playwright: 390px, 768px, 1440px에서 후보가 격자 밖에 유지되고 명시적으로 승격한 뒤에만 배치되는 흐름 통과
- 전체 Playwright: 38개 통과, 환경 조건부 16개 제외
- 390px 후보 흐름에서 문서 가로 overflow 없음
- 기존 자동 접근성 검사 serious/critical 위반 없음

## 다음 확장 경계

후보 바구니가 안정된 뒤 다음 모델을 별도 변경으로 추가한다.

- 여러 과목 중 1개 또는 K개를 선택하는 후보 그룹
- 대표 후보 유형과 `이 시간표와 비슷하게 다시 만들기`
- 자동완성 불가능 조건의 원인별 한 번 완화
- 선택 분반 마감에 대비한 대체 분반 묶음

이 기능들은 후보 바구니 안에서 시작하되, 시간표 캔버스나 전역 내비게이션에 새 책임을 추가하지 않는다.
