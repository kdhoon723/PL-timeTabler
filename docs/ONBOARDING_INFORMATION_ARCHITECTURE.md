# 온보딩 정보구조 재설계

> 상태: 핵심 흐름 구현 · 2026-07-11 UX 검토

## 문제

현재 온보딩은 시간표를 만들려는 사용자에게 `입학연도`, `현재 학년`, `신입학/편입학`, `학생 구분`, `학번 끝자리`를 한 화면에서 묻는다. 이 정보들은 목적이 서로 다르다.

- `학과·전공`, `이번 학기 기준 학년`: 과목 검색과 학년별 전공과목 탐색에 바로 필요
- `입학연도`, `입학 경로`, `학생 구분`: 입학연도별 전공필수·교양필수 규칙을 정확히 적용할 때 필요
- `이수학점`, `이수과목`, `편입 인정학점`: 졸업요건 부족분을 계산할 때만 필요
- `학번 끝자리 분반`: 공식 학과 분반 규칙이 확인된 경우에만 필요

목적이 다른 정보를 한 단계에서 요구하면 재학생은 `신입학/편입학`을 현재 재학 상태로 오해하고, 시간표 작성 전에 불필요한 개인정보를 입력한다고 느낀다.

## 결정

첫 방문 온보딩은 **시간표 작성에 즉시 필요한 최소 정보만** 받는다. 졸업요건과 교과과정 판정 정보는 메인 편집기와 졸업요건 화면에서 선택적으로 추가한다.

### 1. 시간표 기본 설정

첫 방문에만 노출하며 언제든 전체 건너뛰기가 가능하다.

1. `학과·전공 선택`
2. `이번 학기 기준 학년 선택`
3. 바로 편집기로 이동

학년 화면 문구는 `현재 학적 정보를 알려주세요`가 아니라 다음처럼 작업 목적을 설명한다.

- 제목: `이번 학기 몇 학년으로 준비할까요?`
- 설명: `전공 과목을 학년 수준에 맞게 찾는 데 사용해요. 다른 학년 과목도 언제든 검색할 수 있어요.`
- 기본 행동: `{N}학년 시간표 만들기`

입학연도나 입학 경로를 추정해서 저장하지 않는다.

단, 편입생이 일반 학생용 화면을 다시 찾아다니지 않도록 학년 선택 아래에 작은 보조 선택을 둔다.

- 문구: `혹시 편입생인가요?`
- 기본 상태: 선택하지 않음, 편입 관련 설명과 입력은 숨김
- 선택 상태: `대진대 편입학년도`와 인정학점·대체이수는 별도 확인한다는 안내만 펼침
- 일반 학생에게 `신입학/편입학`을 같은 크기의 선택 카드로 보여주지 않음

이 보조 선택은 시간표 진입을 막지 않는다. 선택하지 않으면 `신입학`을 추정하지 않고 학과·학년만 저장한다.

### 2. 학사 기준 설정

온보딩이 아니라 메인 편집기의 필수과목 패널과 프로필 메뉴에서 여는 선택 설정이다.

- 입학연도
- 처음 입학한 경로: 일반 학생은 입학연도 기준을 켜면 `신입학`, 편입생은 첫 방문의 작은 선택 또는 설정 화면에서 `편입학`
- 학생 구분
- 원 입학 학과, 복수·부전공 등 향후 판정에 필요한 항목

`신입학/편입학`은 현재 재학 상태가 아니라 대진대에 처음 들어온 경로라는 설명을 항상 함께 제공한다. 설정하지 않아도 검색·수동 편집·자동 시간표 생성은 모두 사용할 수 있다.

메인 편집기에서는 결과와 연결된 맥락형 행동으로 유도한다.

- 제목: `필수과목 추천을 더 정확히`
- 설명: `입학연도와 입학 경로를 추가하면 적용 가능한 전공필수·교양필수만 확인할 수 있어요.`
- 행동: `학사 기준 설정`

### 3. 이수 현황 입력

`/requirements`의 졸업요건 점검 안에서만 제공한다. 한 번에 전체 성적표를 요구하지 않고 입력 즉시 결과가 좋아지는 순서로 점진적으로 받는다.

1. 총 이수학점·교양학점·전공학점
2. 이수한 필수과목
3. 편입생에게만 인정학점과 대체 인정 항목
4. 향후 사용자 동의가 있을 때만 성적표 파일 불러오기

각 단계에는 `나중에`를 제공한다. 입력하지 않은 값은 `0`이나 충족으로 간주하지 않고 `확인 필요`로 유지한다.

## 데이터 모델 경계

하나의 평면 객체에 모든 의미를 넣지 않고 기본 프로필 안에 선택형 학사 기준을 분리한다. 이수 현황은 졸업요건 영역의 독립 상태로 유지한다.

```text
AcademicProfile
  department
  targetGrade
  academicBasis: AcademicBasis | null

AcademicBasis | null
  admissionYear
  entryRoute
  studentClassification
  originalDepartment
  additionalPrograms

CompletionRecord | null
  totalCredits
  liberalCredits
  majorCredits
  completedCourseIds
  transferRecognitions
```

- `PlanningProfile`: 내 전공 빠른 필터, 학년별 과목 탐색에 사용
- `AcademicBasis`: 입학연도별 필수과목·교양 규칙 적용에 사용
- `CompletionRecord`: 졸업요건 충족·부족 계산에만 사용

저장 스키마는 v2이며, 기존 v1 프로필은 사용자가 직접 입력한 값만 `AcademicBasis`로 옮긴 뒤 기존 키를 제거한다. 새 기본 온보딩에서는 입학연도를 현재 학년으로 역산하거나 `신입학`을 임의 기본값으로 저장하지 않는다.

## 조건부 노출

- 학번 홀짝 분반은 공식 분반 규칙이 확인된 학과에서만 표시한다. 현재 검증 데이터상 적용 학과가 없으면 UI 자체를 숨긴다.
- 편입 인정학점은 입학 경로가 편입일 때만 표시한다.
- 국내학생 전용 교양 규칙은 학생 구분이 확인됐을 때만 판정한다.
- 학사 기준이 없으면 필수과목을 확정하지 않고 학과·학년 기준의 일반 전공 탐색만 제공한다.

## 진입 경로

- 첫 방문: 학과 → 이번 학기 학년 → 편집기
- 메인 필수과목 패널: `학사 기준 설정`
- 상단 프로필: 기본 설정과 학사 기준을 각각 수정
- 졸업요건 페이지: 학사 기준 → 이수 현황 → 점검 결과

## 검증 기준

- 첫 방문 사용자는 큰 `신입학/편입학` 양자택일 없이 편집기에 진입할 수 있다.
- `혹시 편입생인가요?`를 선택하기 전에는 편입 설명과 편입학년도가 보이지 않는다.
- 학과 선택 후 두 번째 화면의 유일한 필수 결정은 이번 학기 기준 학년이다.
- 기본 설정만 저장한 프로필에는 추정 입학연도나 추정 입학 경로가 없다.
- 학사 기준과 이수 현황을 입력하지 않아도 모든 시간표 작성 기능이 동작한다.
- 필수과목 자동 판정은 `AcademicBasis`가 확인된 경우에만 실행된다.
- 편입학년도·학생 구분·분반 홀짝 필드는 적용 가능한 상황에서만 나타난다.
- 390px 화면에서 기본 온보딩은 스크롤 없이 핵심 설명·학년 선택·기본 행동이 보인다.
