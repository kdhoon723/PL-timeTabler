# Security Policy

## Supported version

보안 수정은 현재 `main` 브랜치와 운영 중인 최신 배포에 적용합니다.

## Reporting a vulnerability

민감한 내용을 공개 이슈에 게시하지 마세요. GitHub 저장소의 **Security → Report a vulnerability**를 통해 비공개로 제보해 주세요.

제보에는 가능한 범위에서 다음 내용을 포함해 주세요.

- 영향을 받는 URL, API 또는 컴포넌트
- 재현 단계와 기대 동작
- 노출될 수 있는 데이터 또는 권한 범위
- 이미 시도한 완화 방법

재현에 필요한 최소 정보만 공유하고 실제 학생 계정, OTP, 세션 쿠키, API 키와 비밀번호는 포함하지 마세요. 확인 후 영향 범위와 수정 계획을 비공개 채널에서 안내합니다.

## Public issues

일반 버그와 기능 제안은 GitHub Issues를 사용해도 됩니다. 로그나 화면 캡처를 첨부할 때 학번, 이메일, 시간표 공유 URL 등 개인 식별 가능 정보를 제거해 주세요.
